Release notes for iteraplan 3.3.1 Final
=========================================

Fixed bugs (vs. 3.3.0):

* Fixed an issue which occurred, when trying to apply a filter on Relationship-End with IteraQL.
* Fixed an issue preventing successful imports if there were values missing for mandatory attributes in Business Mappings.
* Update of multivalued attributes in interfaces is now possible.
* Fixed an issue when using the query filter on building block overview pages.
* Fixed an issue causing errors with Export/Import and Power Queries, if attributes had invalid names.
* Fixed issue which allowed the creation of attributes and attribute values with invalid or conflicting names.
* Fixed an issue preventing the re-import of a partial export using enumeration attribute values in its filter predicate
* Fixed an issue preventing iteraplan to run correctly when installed in Tomcat's ROOT context.
* Fixed an issue which lead to a permanent opened coloring dialog for Nesting Cluster Diagram.
* Fixed misleading error message during file downloads.
* Fixed issue allowing the creation of building blocks with duplicate names when using certain database configurations.
* Fixed an issue which sometimes prevented successful iteraplan installation using console (headless) mode.
* Fixed an excel import issue with elements of type Isr2BoAssociation, Tcr2IeAssociation and InformationFlow which prevented their creation or deletion.

Improvements/new features (vs. 3.3.0):

* Added a default color to enumeration attribute values added by the Excel/XMI import.
* Improved error messages and error handling in some import cases (both Excel and XMI).
* Response status codes in iteraplan's REST API are now more meaningful.
* The output format "CSV" is now supported for direct list export in Tabular Reporting.

Release notes for iteraplan 3.3.0 Final
=========================================

New features (vs. 3.2):

* Major overhaul of iteraplan's export & import capabilities
	** partial export/import: possibility to export and import only a specific subset of all data
	** Different strategies for import: Additive (Create/part. Update) & Overwrite (Create/Update/Delete)
	** REST interface: Write access to single building block elements (Create/Update/Delete)
	** HTTP Basic Authentication for REST-API access
* New direct Excel export: WYSIWYG-export of simple-list query results or spreadsheet reports
* Report type independent overview for all saved queries
* Added Kerberos-based Single Sign On in combination with Microsoft IIS
* Tested for Oracle 12 compatibility
* Backwards 3.x compatibility with Excel import improved
* LDAP-based authentication: given and last names are synchronized at login time
* Improved IE10 support
* Improved error reporting capabilities 

Fixed bugs (vs. 3.2):

* Fixed issue with displaced text on attribute description popups
* Fixed issue with missing connection lines in information flow diagrams
* Fixed a bug causing an error page during configuration of spreadsheet reports. It was related to certain business mapping queries.
* Fixed a bug preventing the deletion of infrastructure elements in certain cases
* Fixed issue with notifications not being sent for certain status changes
* Fixed issue with switching the language on the query console page
* Fixed issue with changing enum attribute values using Internet Explorer
* Fixed issue with deleting an infrastructure element
* Various minor improvements and fixes

Distinguishing features in the Enterprise Edition 
==========================================================

Following features are exclusively available in the Enterprise Edition of iteraplan, 
and are not included in the Community Edition: 

- Support for MySQL and Oracle Database storage (while Community Edition is restricted to HSQLDB)
- User management with roles and permissions
- Supporting queries
- Integration capability with an LDAP infrastructure for central Authentication and Authorization
- Import data from Excel and XMI files
- Visio-Helper (Infoflow-Layout & Impact-Analysis)
- Power-Queries (iteraQL)

== Release-Übersicht für iteraplan 2.2 ==
=========================================

Neue Funktionen und Änderungen (gegenüber Version 2.1)
======================================================

Grafische Auswertungen können gespeichert werden. Für alle Grafiken können Suchanfragen 
mit den spezifischen Einstellungen gespeichert und gelöscht werden (sofern die Rechte 
dafür eingeräumt werden). Somit können wiederkehrende Abfragen hinterlegt werden und 
müssen nicht mühsam eingegeben werden. Die gespeicherten Abfragen sind dabei für alle 
Benutzer sichtbar. 
 
Die Informationsfluss-, Bebauungsplan-, Portfolio- und Masterplan-Grafiken wurden 
angeglichen. In diesen Grafiken ist ein einheitliches Vorgehen umgesetzt. Dies bezieht 
sich auf die Suchanfragen, den Workflow und die farbliche Gestaltung von Merkmalen. Es 
können Anfrageerweiterungen benutzt werden und man kann Ergebnisse nachbearbeiten. Nach 
Auswahl der Ergebnissmenge wird ein zweiter Schritt geladen, in dem Farben für Merkmale 
gewählt und Abfragen gespeichert werden können. Für alle Grafiken stehen nun auch Merkmale 
zur Codierung von Farben zur Verfügung. Der Status (von Informationssystemen und 
Technischen Bausteinen) wird als normales Merkmal aufgelistet und steht nun in allen 
Grafiken zur Verfügung. Mögliche Farben können über Einstellungsdateien definiert werden.

Numerische Attribute können für Grafische Auswertungen in Wertebereiche eingeteilt werden. 
Hierzu wurde die Definition der Zahl-Merkmale erweitert. Es kann gewählt werden, zwischen 
einer Gleichverteilung aller Merkmalsausprägungen und benutzerdefinierten Grenzen.

In Tabellarischen Auswertungen sind neue Merkmale hinzugekommen. Es kann zusätzlich nach 
'Letzter Benutzer' bzw. 'Letzte Änderung' gesucht werden. Somit können Änderungen im System 
schnell visualisiert werden, Falls das sogenannte Last-Modification-Logging eingeschalten 
ist. 
 
Die Version wurde internationalisiert. Momentan stehen eine deutsche und eine englische 
Version zur Verfügung. 
 
Neben diversen Qualitätsverbesserungen wurden noch folgende Funktionen ergänzt: 
Excel-Export ist nun für alle Bebauungselemente verfügbar. 
Der Umfang der Konsistenzchecks wurde erweitert.
Kleinere Verbesserungen an der Benutzeroberfläche
Firefox 3 Kompatibilität
Behebung von kleineren Fehlern

Diverse Java-Bibliotheken wurden aktualisiert.

== Release-Übersicht für iteraplan 2.1 ==
=========================================

Neue Funktionen und Änderungen (gegenüber Version 2.0)
======================================================

Die Pflege der fachlichen Bebauung wurde um die Konzepte 'Fachliche Funktionen' und 'Fach-
liche Domänen' erweitert. Diese sind vollständig in sämtliche Bereiche von iteraplan inte-
griert, z.B. tabellarische/grafische Auswertungen und Massenupdate. 

  Fachliche Funktionen: Eine fachliche Funktion ist eine in sich abgeschlossene und zusam-
  menhängende fachliche Tätigkeit, wie beispielsweise 'Kunde anlegen' oder 'Produktkonfi-
  guration verändern'. Fachliche Funktionen können den Informationssystemen direkt zugeord-
  net werden. Dies ermöglicht die Modellierung der fachlichen Funktionalität von Informa-
  tionssystemen. Zudem ist eine Zuordnung von Geschäftsobjekten zu fachlichen Funktionen 
  modellierbar.

  Fachliche Domänen: Fachliche Domänen sind fachliche Einteilungen. Sie strukturieren die 
  fachliche Bebauung. Die fachliche Bebauung beschreibt die wesentlichen fachlichen Ein-
  heiten eines Unternehmens, die maßgeblich das Geschäft bestimmen. Die wesentlichen Ele-
  mente der fachlichen Bebauung sind Geschäftsprozesse, fachliche Funktionen, Produkte, 
  Geschäftseinheiten und Geschäftsdaten.
    
Die Terminologie von iterplan wurde weiter verfeinert und die folgenden Begriffe innerhalb 
der Anwendung konsistent umbenannt:
 
 IT-Vorhaben -> Projekte
 Organisationseinheiten -> Geschäftseinheiten
 Business-Unterstützung -> Fachliche Zuordnung

Die Masterplan-Grafik wurde um die Möglichkeit erweitert, nicht nur 'Informationssysteme' 
und 'Technische Bausteine' anzuzeigen, sondern auch 'Projekte'. Im weiteren Verlauf kann 
bei der Auswahl von Projekten die Option 'Hierarchisch' dazu genutzt werden, den Pro-
jekten zugeordnete Informationssysteme ebenfalls in der Grafik anzeigen zu lassen.

Für die Erzeugung einer Portfolio-Grafik steht die neue Option 'Skalierung' zur Verfügung.
Dadurch kann die Betrachtung einer Portfolio-Grafik auf einen lokalen Ausschnitt begrenzt 
werden. Dieser wird durch die in der Grafik enthaltenen Elemente bestimmt, d.h. die maxi-
male bzw. minimale Ausprägungen eines Merkmals in der Grafik bilden die Achsenbegrenzung.

Die Pflege der Basisdaten 'Infrastrukturelement', 'Geschäftsobjekt', 'Projekt' sowie der 
neu hinzugekommenen Elemente 'Fachliche Funktion' und 'Fachliche Domäne' erfolgt im Gegen-
satz zu iteraplan 2.0 nun mittels geordneter Hierarchien. Mit diesem Release sind daher 
alle Ausprägungen von Basisdaten innerhalb geordneter Hierarchien pflegbar.

Die neue Einstiegsgrafik veranschaulicht die Gesamtheit aller möglichen Beziehungen zwischen 
den Bebauungselementen und legt besonderen Wert auf die Einstufung dieser in die verschiedenen 
Bereiche des Bebauungsmanagements 'Fachliche Bebauung', 'Informationsbebauung', 'Technische 
Bebauung', 'Betriebsinfrastruktur-Bebauung' sowie 'Produktportfolio'. Über einheitliche 
Symbole werden nun neben den Beziehungen zwischen den Modellelementen auch Aspekte wie
z.B. hierarchische Strukturierungen und Vorgänger/Nachfolger-Relationen abgebildet.

Die Möglichkeit der Vererbung von Merkmalen auf Teil-Bebauungselemente wurde entfernt, da
identische Merkmalswerte für hierarchische Strukturen nun auf einfache Weise über das
Massenupdate gepflegt werden können.

Darüber hinaus wurde sehr viel Wert darauf gelegt, eine Optimierung der Darstellung der 
grafischen Auswertungen zu erreichen, vorhandene Unstimmigkeiten und kleinere Fehler zu be-
seitigen sowie insgesamt Performance und Stabilität der Anwendung zu erhöhen.
